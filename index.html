QUESTION:

As a Natural Language Processing (NLP) analyst, you are tasked by your supervisor to design a function which would take a list of string variables as input then output a bag-of-word dictionary.

Input:

  • Your input data would always be a list of string variables

  • Your input data would not be an empty list

  • There might be scenarios where there are leading/ trailing spaces in your input data, e.g. ' a' or 'b '. However, your bag-of-word dictionary output would have these spaces removed.

  • There might be scenarios where your input data is having some uppercase character(s), e.g. 'Apple' or 'baNaNa'. However, your bag-of-word dictionary output should only consist of all lowercase characters.

  • There would not exist any non-word characters, e.g. '!','@'

  • The maximum length of the input list would be capped at 10,000

Example 01:

Given an input data of [ ' a' , 'b ' , 'Apple' , 'baNaNa' , 'orange' , 'orange' ], your output should be:

Python=> {'a': 1, 'b': 1, 'apple': 1, 'banana': 1, 'orange': 2}

Javascript=> {a: 1, b: 1, apple: 1, banana: 1, orange: 2}

R=>

a b apple banana orange
1 1 1 1 2

Python=> {'invoice': 1, 'receipt': 2, 'fund': 1, 'accounts': 1, 'account': 1}

Javascript=> {invoice: 1, receipt: 2, fund: 1, accounts: 1, account: 1}

R=>

invoice receipt fund accounts account
1 2 1 1 1
